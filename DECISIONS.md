# Decision Log

## Format
```
[YYYY-MM-DD HH:MM] - Decision Title
Context: Why this decision was needed
Decision: What was decided
Rationale: Why this choice
Impact: What this affects
```

---

## Decisions

### Example
```
[2026-01-31 12:15] - Use Mock Data for Demo
Context: API integration taking too long
Decision: Hardcode demo data in frontend
Rationale: Demo stability > real API
Impact: Tasks 04-05 scope reduced
```

---

**Purpose:** Track all scope changes, tech choices, and cuts. This prevents revisiting decisions under pressure.
